1. PROJECT OVERVIEW
================================================================================

PROJECT NAME: Unsupervised Network Monitoring and Botnet Detection System

DESCRIPTION:
A hybrid botnet detection system that analyzes network traffic from PCAP files
using machine learning to identify malicious botnet activity. The system 
operates in two modes:
  - Offline Mode: Direct PCAP analysis with file output
  - Streaming Mode: Real-time simulation with live dashboard

TECHNOLOGY STACK:
  - Backend: Python, FastAPI, SQLite
  - ML Models: Scikit-learn (KMeans + Random Forest)
  - Streaming: Apache Kafka
  - Frontend: React, Vite, Recharts
  - Infrastructure: Docker Compose

KEY FEATURES:
  ✓ Real-time botnet detection using ML
  ✓ Flow-based network traffic analysis
  ✓ Live streaming dashboard with charts
  ✓ PCAP file upload via web interface
  ✓ Host-level aggregation (5% threshold)
  ✓ Hybrid ML approach (clustering + classification)

3. COMPONENTS DESCRIPTION
================================================================================

3.1 DETECTOR MODULE (detector/)
--------------------------------
Purpose: Core ML detection logic

Files:
  - feature_extractor.py: Parses PCAP, builds flows, extracts 30 features
  - predictor.py: Loads models, preprocesses data, runs predictions
  - host_aggregator.py: Aggregates flow predictions to host-level
  - offline.py: Orchestrates offline detection workflow

Key Features Extracted (30 total):
  1. total_pkts - Total packets in flow
  2. null_pkts - Packets with no payload
  3. small_pkts - Packets under 5KB
  4. percent_of_sml_pkts - Percentage of small packets
  5. total_duration - Flow duration in microseconds
  6. average_payload - Mean payload size
  7. stddev_packet_length - Standard deviation of packet lengths
  8. freq_packet_length - Most frequent packet length ratio
  9. payload_per_sec - Payload bytes per second
  10. avg_inter_times - Average inter-arrival time
  11. med_inter_times - Median inter-arrival time
  12. var_packet_size - Variance of packet sizes
  13. max_packet_size - Maximum packet size
  14. average_packet_length - Mean packet length
  15. first_packet_length - First packet length
  16. average_packet_ps - Packets per second
  ... and more

ML Models:
  - KMeans Clustering: Identifies obvious benign/botnet patterns
    * Clusters 1,2,5,8 → Benign
    * Cluster 6 → Botnet
    * Other clusters → Random Forest decides
  
  - Random Forest Classifier: Handles ambiguous flows
    * 200 estimators
    * Trained on labeled botnet datasets

Host Aggregation Rule:
  - If ≥5% of flows from a host are botnet → Host marked as Botnet
  - Otherwise → Host marked as Benign

3.2 PRODUCER MODULE (producer/)
--------------------------------
Purpose: Simulates live traffic from PCAP files

File: pcap_replay_producer.py

Features:
  - Reads PCAP in time windows (default: 5 seconds)
  - Extracts flow features using detector module
  - Publishes to Kafka topic: flow_features
  - Configurable replay delay (default: 1 second)

Usage:
  python -m producer.pcap_replay_producer <pcap_file> [--replay-delay 0.5]

3.3 CONSUMER MODULE (consumer/)
--------------------------------
Purpose: Processes flow features and generates predictions

File: detection_consumer.py

Features:
  - Consumes from Kafka topic: flow_features
  - Loads ML models from Models/ directory
  - Runs prediction on each flow
  - Maintains host-level aggregation
  - Publishes to topics: predictions, alerts

Usage:
  python -m consumer.detection_consumer

3.4 BACKEND MODULE (backend/)
------------------------------
Purpose: REST API and data storage

Files:
  - main.py: FastAPI application with endpoints
  - database.py: SQLite database operations
  - kafka_listener.py: Background Kafka consumers

Database Tables:
  1. flows: Individual flow predictions
     - id, flow_id, timestamp, src_ip, dst_ip, sport, dport, protocol, prediction
  
  2. hosts: Host-level aggregation
     - src_ip, total_flows, benign_flows, botnet_flows, botnet_percentage, status
  
  3. alerts: Security alerts
     - id, timestamp, src_ip, status, botnet_percentage, reason

API Endpoints:
  GET  /api/summary       - Overall statistics
  GET  /api/flows         - Recent flow predictions
  GET  /api/hosts         - Host monitoring data
  GET  /api/alerts        - Recent alerts
  GET  /api/timeseries    - Time-series data for charts
  GET  /api/protocols     - Protocol distribution
  POST /api/upload-pcap   - Upload PCAP file
  POST /api/sync-database - Trigger database sync
  DELETE /api/clear-data  - Clear all data

Usage:
  uvicorn backend.main:app --reload

3.5 DASHBOARD MODULE (dashboard/frontend-app/)
-----------------------------------------------
Purpose: Web-based visualization interface

Files:
  - src/main.jsx: React application
  - src/styles.css: Styling
  - index.html: HTML template
  - package.json: Dependencies

Features:
  - 6 statistics cards (total flows, benign, botnet, hosts, alerts)
  - Live flow stream table
  - Host monitoring table
  - 3 charts: Flows over time, Prediction mix, Protocol distribution
  - Alerts panel
  - PCAP upload interface
  - Auto-refresh every 2 seconds

Usage:
  cd dashboard/frontend-app
  npm install
  npm run dev

Access: http://localhost:5173/

3.6 STREAMING MODULE (streaming/)
----------------------------------
Purpose: Kafka utilities

File: kafka_utils.py

Functions:
  - create_producer(): Creates Kafka producer with JSON serialization
  - create_consumer(): Creates Kafka consumer with JSON deserialization

3.7 MODELS DIRECTORY (Models/)
-------------------------------
Purpose: Pre-trained ML models

Files:
  - cluster.pkl: KMeans clustering model
  - flow_predictor.joblib: Random Forest classifier
  - label_encoder.pkl: Protocol label encoder
  - mms.pkl: MinMaxScaler for feature normalization

Training Data:
  - Zeus botnet traffic
  - Vinchuca botnet traffic
  - Benign P2P traffic (torrents)

9. TROUBLESHOOTING
================================================================================

9.1 COMMON ISSUES
-----------------

Issue: Port 8000 already in use
Solution:
  # Find process using port
  netstat -ano | findstr :8000
  
  # Kill process
  taskkill /F /PID <process_id>

Issue: Kafka connection failed
Solution:
  # Restart Kafka
  docker compose restart
  
  # Check Kafka status
  docker ps

Issue: Dashboard shows zeros
Solution:
  # Manually sync database
  python quick_sync.py
  
  # Or use dashboard "Sync Database" button

Issue: Upload fails
Solution:
  # Check if consumer is running
  python -m consumer.detection_consumer
  
  # Check file extension (.pcap or .pcapng only)

Issue: Database locked
Solution:
  # Stop backend
  # Delete database
  Remove-Item backend/botnet_stream.db
  
  # Restart backend
  uvicorn backend.main:app --reload

Issue: Windows Kafka error "Invalid file descriptor: -1"
Solution:
  # This is a known kafka-python + Python 3.13 + Windows issue
  # Use manual sync instead:
  python quick_sync.py

9.2 VERIFICATION STEPS
----------------------

Check Backend:
  curl http://localhost:8000/api/summary
